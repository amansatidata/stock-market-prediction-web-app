# Contributing to Stock Market Prediction App

First of all, thank you for considering contributing to this project! 🎉

Contributions help improve the project, fix bugs, enhance functionality, and make the application more useful for the community. Whether you're reporting issues, improving documentation, fixing bugs, or implementing new features, your contributions are highly appreciated.

---

## 📋 Code of Conduct

By participating in this project, you agree to:

* Be respectful and professional in all interactions.
* Provide constructive feedback.
* Welcome new contributors and support collaborative development.
* Focus on improving the project for everyone.

---

## 🚀 Getting Started

### 1. Fork the Repository

Click the **Fork** button at the top-right corner of the repository page.

### 2. Clone Your Fork

```bash
git clone https://github.com/amansatidata/Stock-Market-Prediction.git
cd Stock-Market-Prediction
```

### 3. Create a Virtual Environment

```bash
python -m venv venv
```

Activate the environment:

**Windows**

```bash
venv\Scripts\activate
```

**Linux/macOS**

```bash
source venv/bin/activate
```

### 4. Install Dependencies

```bash
pip install -r requirements.txt
```

### 5. Run the Application

```bash
streamlit run app.py
```

## 🛠 Ways to Contribute

You can contribute by:

### 🐛 Reporting Bugs

When reporting bugs, please include:

* Operating System
* Python Version
* Error Message
* Steps to Reproduce
* Expected Behavior
* Screenshots (if applicable)

---

### ✨ Suggesting Enhancements

Enhancement suggestions may include:

* New forecasting models
* Performance improvements
* UI/UX improvements
* Additional stock market features
* Improved data visualization

Please open an Issue before starting major feature development.

---

### 💻 Submitting Code Changes

1. Create a new branch:

```bash
git checkout -b feature/your-feature-name
```

2. Make your changes.

3. Commit your changes:

```bash
git commit -m "Add: meaningful description of changes"
```

4. Push to your fork:

```bash
git push origin feature/your-feature-name
```

5. Open a Pull Request.

---

## 📏 Development Guidelines

### Code Quality

* Follow Python best practices (PEP 8).
* Use meaningful variable and function names.
* Write clean and maintainable code.
* Keep functions modular and reusable.

### Documentation

* Update documentation when introducing new features.
* Include comments where necessary.
* Ensure README instructions remain accurate.

### Testing

Before submitting a Pull Request:

* Verify the application runs successfully.
* Check that visualizations load correctly.
* Confirm forecasting functionality works as expected.
* Ensure no existing features are broken.
