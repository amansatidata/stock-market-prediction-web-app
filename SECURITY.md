# Security Policy

## 🛡️ Supported Versions

The following table indicates which versions of the project currently receive security updates and support.

| Version           | Supported          |
| ----------------- | ------------------ |
| Latest Release    | ✅ Yes              |
| Previous Releases | ⚠️ Limited Support |
| Older Releases    | ❌ No               |

Users are encouraged to use the latest version of the application to receive security improvements and bug fixes.

---

## 📢 Reporting a Vulnerability

The security of this project is taken seriously. If you discover a security vulnerability, please report it responsibly.

### Please Do Not

* Open a public GitHub issue for security vulnerabilities.
* Publicly disclose the vulnerability before it has been reviewed and addressed.
* Share exploit details that could put users at risk.

### Please Include

When reporting a vulnerability, provide the following information:

* A clear description of the issue.
* Steps to reproduce the vulnerability.
* Potential impact and severity.
* Screenshots or proof-of-concept (if applicable).
* Suggested remediation (optional).

## 🔍 Response Process

After receiving a security report, the following process will be followed:

1. Acknowledge receipt of the report.
2. Investigate and validate the vulnerability.
3. Assess impact and affected components.
4. Develop and test a fix.
5. Release a patched version.
6. Publicly disclose the issue after remediation when appropriate.

---

## ⏱️ Response Timeline

| Action                   | Target Time         |
| ------------------------ | ------------------- |
| Initial Acknowledgement  | Within 7 Days       |
| Vulnerability Assessment | Within 14 Days      |
| Fix Development          | Based on Severity   |
| Security Release         | As Soon As Possible |

Response times may vary depending on the complexity and severity of the reported issue.

---

## 🔐 Security Best Practices

Users of this project are encouraged to:

* Keep dependencies updated.
* Use supported Python versions.
* Review third-party packages before deployment.
* Regularly monitor dependency vulnerabilities.
* Avoid exposing sensitive credentials in source code.

---

## ⚠️ Disclaimer

While every effort is made to maintain the security and reliability of this project, no software can be guaranteed to be completely free of vulnerabilities. Users should evaluate the software according to their own security requirements and deployment environments.

---

## 📄 Scope

This security policy applies to:

* Source code contained within this repository.
* Project dependencies and integrations.
* Application deployment configurations.

This policy does not apply to vulnerabilities originating from third-party services such as:

* Yahoo Finance (yFinance)
* Streamlit Cloud
* Prophet
* Plotly
* Other external libraries and APIs

Users should refer to the respective vendors for security-related concerns involving third-party services.
